## 1. Deployment / system context

```mermaid
flowchart TB
  user([User browser / PWA])

  subgraph Vercel["Vercel (web prod)"]
    spa["bujo SPA (Vite/React)\nbujo-journal.vercel.app"]
    fn["/api/feedback (serverless)"]
  end

  subgraph SelfHost["Self-host (docker compose)"]
    web["web — nginx + SPA :8080"]
    api["api — PostgREST :3000"]
    db[("db — Postgres :5432")]
    adminer["adminer :8081"]
  end

  supa[("Supabase\n(optional account sync)")]
  gist[("GitHub Gist\n(optional backup)")]
  folder[("Local cloud folder\n(File System Access)")]

  user --> spa
  user --> web
  spa -. account .-> supa
  spa -. gist token .-> gist
  spa -. folder .-> folder
  web --> api --> db
  adminer --> db
  spa -. self-host URL .-> api

  classDef store fill:#313244,stroke:#cba6f7,color:#cdd6f4;
  class db,supa,gist,folder store;
```

bujo is **local-first**: the source of truth is the in-browser `JournalData`. Every backend (Vercel, Supabase, self-host PostgREST, gist, folder) is an *optional* sync target. With E2E, those targets hold ciphertext.

## 4. Sync sequence (save → persist → flush on close)

```mermaid
sequenceDiagram
  participant U as User
  participant V as View
  participant S as store (useJournal)
  participant LS as localStorage
  participant SS as ServerSync
  participant API as PostgREST /journals

  U->>V: tap (e.g. check a habit)
  V->>S: patch(data)
  S->>LS: write JournalData (immediate)
  S-->>SS: data changed
  SS->>SS: debounce 2.5s
  SS->>API: POST /journals (upsert blob)
  API-->>SS: 201
  Note over U,SS: on tab close
  U->>SS: pagehide / visibilitychange=hidden
  SS->>API: POST /journals (keepalive flush)
```

## Regenerating
These are hand-maintained from the code. When the data model, component tree, or sync paths change, update the relevant diagram in the **same PR** — `docs/RULES.md` makes this a checklist item, and `.github/workflows/docs-guard.yml` flags code-only PRs.
